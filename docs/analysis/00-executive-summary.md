## 한 줄 결론

아이디어의 빈틈은 실재하지만, **지금 코드는 사업계획서와 다른 물건이다.** 핵심 차별점 3개(AI 브리핑·영/한 이중언어·수익화)가 전부 미구현이라 현 상태 수익 확률은 ~15%다. 다만 셋 다 "신규 개발"이 아니라 이미 있는 뼈대에 **배선만 하면 되는 작업**이라, 90일 안에 배선을 끝내고 웨지 시장을 검증하면 확률이 ~40%까지 오른다. **판정: 조건부 GO.**

## 2. 관점별 핵심 판정

### 개발자(CTO)
- **잘한 것(시니어급):** DDD 컨텍스트 분리 실제 준수(~8,900 LOC, 1인 유지에 이상적), RLS 80개 정책·해시 토큰·SecureStore 세션으로 보안 성숙, `npm run verify` 게이트.
- **P0(출시 전 필수):** 엣지 함수 배선(완성된 백엔드 연결), 결제(RevenueCat + Plus 트라이얼), 크래시 리포팅+ErrorBoundary, 스토어 심사물(**Apple 앱 내 계정삭제 필수 — 없으면 리젝**, 개인정보처리방침 URL).

### 사장(CEO)
- **수익성:** 순수 기록 구독 ~15% → 이중언어 웨지+실제 LLM+커머스/보험 레이어 ~40%.
- **두 시장 모두 순수 구독은 사망 경로.** 호주 프리미엄 전환 median 2.1%, 한국은 순수 기록 앱 무료가 기본값(흑자는 전부 커머스 결합).
- **단위경제는 훌륭**(LLM 붙여도 유저당 연 원가 US$0.5~1.5, 마진 95%+). 문제는 원가가 아니라 전환.

### 마케터
- **돈은 P1(아픈 반려동물 보호자)에게서 나온다.** 지불 트리거 = 진단 직후 2주. 예산 80% 집중.
- **성장 루프가 하나도 실존하지 않음**(가족 초대 K=0, vet 링크 웹 뷰어 부재로 바이럴 0). 마케팅보다 루프 구축이 선행.
- **온보딩 계정 강제**가 상단 퍼널 대량 이탈 유발 → 기존 이중 모드로 "가입 전 체험" 재배선 권장.

## 3. 이 앱만의 유니크함 (최종)

개별 기능(가족 기록·AI vet 리포트·링크 공유·한국 병원기록)은 전부 경쟁자가 있다(Petmaid, CompanAIn, Omelo, 촉촉한코/우리엔왈). 방어선은 기능이 아니라:

> **"반려동물 건강의 종적 원장(longitudinal ledger)".** 경쟁사 데이터는 병원 방문이라는 이벤트에서 나와 띄엄띄엄하고 병원에 종속된다. PawBloom은 매일의 식사 g·물 ml·배변·컨디션을 직접 쌓으므로 (a) 이상탐지 baseline이 촘촘해지고, (b) 정량 시계열은 언어 중립적이라 영/한 무손실 렌더되며, (c) 병원·국가를 옮겨도 원장이 따라온다. **유니크함은 기능이 아니라 "시간이 만드는 데이터 밀도"** — 오늘 복제해도 내일 따라잡을 수 없는 유일한 자산.

- **가장 뾰족한 타깃:** "만성질환 + 이동(이민·이사·다병원) 가능성이 있는 보호자" — 종적 데이터·소유권·이중언어 3자산이 동시에 필요한 유일한 세그먼트.
- **두 가지 전제:** 데이터 export가 있어야 "소유권" 주장이 진실이 됨(현재 없음), 일일 기록 리텐션이 유지돼야 원장이 끊기지 않음(리마인더 설계 = moat의 생명선).

## 4. 교차검증에서 드러난 사각지대 리스크

1. **호주 TGA 의료기기 소프트웨어(SaMD) 규제** — LLM 브리핑이 증상→행동 권고를 생성하면 규제 경계 근접. "6주 내 LLM 출시" 권고는 규제 검토 병행 필수. disclaimer 한 줄로 끝나지 않음.
2. **수의사 수용성 미검증** — vet 공유가 moat 핵심인데 수의사가 보호자 자가기록+AI 요약을 신뢰할지 미검증. 베타 때 vet 인터뷰를 검증 게이트에 포함.
3. **데이터 export 부재** — "내가 소유하는 기록"이 마케팅 거짓말이 되는 지점. 호주 개인정보법 흐름상 강제될 수도.
4. **계절성** — 호주 펫보험·입양은 12~2월(여름) 피크. 보험 리드 수익·출시 타이밍을 이 시즌에 앵커하면 CAC가 달라짐.

## 5. 종합 실행 우선순위

| 순위 | 액션 | 기간 | 근거 |
|---|---|---|---|
| 1 | 언어 스위처 + `t(lang)` 배선 + 인라인 한국어 이관 | 1~2주 | 모든 전략의 전제. 이미 있는 en 번역 배선 |
| 2 | 엣지 함수 배선 + 실제 공유 웹 뷰어 | 2~3일+α | 백엔드 완성. 유일하게 실재하는 차별점을 살림 |
| 3 | 체크리스트 토글/night-memo 버그 + 계정삭제 + Sentry + 퍼널 계측 | 1~2주 | 신뢰·심사·측정의 기반 |
| 4 | LLM 브리핑 실물화 (규제 톤 검토 병행) | 2~3주 | 데이터 파이프라인 완성, 호출부만 교체 |
| 5 | 가족 초대 딥링크 + 로컬-우선 온보딩 + 저녁 리마인더 | 2~3주 | 성장 루프·리텐션 훅 생성 |
| 6 | RevenueCat + Plus 트라이얼 + 베타 200명 검증 | ~12주차 | Go/Kill 게이트 |

### 90일 Go/Kill 게이트
- **G1 (3주):** 이중언어 실동작 — 영어 100% 렌더
- **G2 (6주):** 진짜 LLM 브리핑 출시 (규제 톤 검토 병행)
- **G3 (12주):** 베타 200명에서 트라이얼→유료 전환 ≥12% 또는 보험/커머스 리드 클릭 ≥8%

하나라도 실패 시 Kill 재검토. Pivot 방향: 축적된 정량 케어 데이터 파이프라인을 B2B(병원·보험·펫푸드)에 판매.

> 수치(확률 15~40%, TAM 계수, 전환율 벤치마크)는 정밀값이 아니라 방향성 추정이며 일부는 상호인용된 미검증 값이다. 인용·의사결정 시 각 문서의 출처·가정을 함께 확인할 것.
